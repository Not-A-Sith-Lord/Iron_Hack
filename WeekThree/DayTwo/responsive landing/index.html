The Three Tenants of Responsive Design

Media Queries

Media Queries are the bread and butter of responsive design. CSS isn't considered a real programming languge (but apparently it might be Turing Complete?) so I was a bit surprised when I first discovered Media Queries, which are basically your typical "If-Then" programming statement within a CSS stylesheet. Using these conditionals you can create what are called "breakpoints" in a webpage.

Simply put, you can tell a browser that when the viewport reaches a certain width, say 1080 pixels, then it should apply certain styling then and only then. This allows you to use a "mobile first" approach. You begin by coding for the smallest screen size, then by placing breakpoints at appropriate viewport widths you can apply additional styling that reshapes the HTML into a progressivly larger screen experince. This allows you to build for any concievable screen size using just one HTML page and a single CSS page. Pretty nifty.

This page you're currently reading posseses three main breakpoints. As you shrink and stretch the browser window you may spot them by discrete changes in how the content is formatted. As you've noticed the fixed display at the bottom left of your screen shows the current Media Query. Any variation in screen size within those Media Queries is handled by the next two tenants of responsive design.

Flexible Images and Media

I was never a big fan of math in school, but if there was one aspect that peaked my interest it was geometry. Shapes were something I could easily vizualize and manipulate in my head, and this kind of problem solving's come back to me now when building responsive web pages. Rather than thinking about how a web page should look in fixed measurements, I'm forced to think about it in more abstract relative and porportional terms. How large do I want this image element to be in relation to this text element for example, and how do I code that so that both scale like I want them to?

Flexible media was the first piece of this puzzle. Flexible media demands that you ditch fixed measurements, like an image having a width of 400 pixels. Instead you would set this measurement to a percentage, such as 40% of the parent HTML container. Containers adjust their size continually in porportion to the browser window, so by setting the width of the image in relation to one you have a responsive image. This applies to page elements of any kind, although to be sure not everything has to or should be responsive in this way.

Flexible Image

While simple enough to implement this technique with any single element, on a crowded webpage I found things could get exponentially complicated quickly if everything is sized in relation to everything else. This made maintaning clean CSS and readable HTML pivotal, as well as picking a choosing when this is the best choice. But it also forced me to mentally visualize the ratios and relationships between all the "shapes" on my webpage, just like back in geometry.

Flexible Grids

In order to accomodate any unqiue screen size within our seperate device experinces (established by our Media Queries), we have to expand on the flexible media concept. Rather than apply porportional values to just individual page elements, we can nest these elements into different sections of a grid and have these sections scale according to the specific screen size. This flexible grid works to fill in any additional space available on whatever device a visitor happens to be using, or shrink space accordingly. Some Additional Tools and Tricks

Besides the three main tenants, there's various other tools that can be used to further enhance page responsivness.

Rems

Unlike pixels, rems are a unit of measurement that are dependent on the default font size of a page. If the default font size is 16px, then 1 rem = 16px. This is helpful since every browser has its own default font size according to the type of device it is on. A desktop browser may have 16px as its default, and a phone browser might have 11px to account for the smaller screen. So by setting all fixed units to rems rather than pixels, font sizes and other elements will automatically tweak themselves in relation to the browser default on the visitors device.

Srcset

Srcset allows you to declare a set of images to be displayed on different screen resolutions. Perhaps you have a very large and high resolution image that looks great for retina desktop users, but for mobile users you'd like them to see a more cropped and low resolution version of the same image. Srcset allows you to do that by having the browser determine which image resolution can actually be seen on the visitor's device. This has the benefit of not wasting bandwith on a high resolution image that will make no noticable difference on the visitors screen.

Div and Background Image

Instead of placing an image in the HTML with an "" tag, you can place a div instead and set the desired image as its background. This has the advantage of being able to swap out that image with more or less cropped versions depending on viewport size using our Media Queries. I demosntrate this with the HTML and CSS logo at the top of this page.

Set Display Property to None

Media Queries allow you to apply radically different stlying (or not) to create different experinces that only take effect at certain screen sizes. This can mean just changing the dimensions of page elements and moving them around. However often times as the screen size gets much smaller it makes sense to get rid of non-essential elements entirly. Simply set the display property for these elements to "none" in the desired section of the CSS (marked by the Media Query conditional) to make it dissapear, and override this when you feel the screen is large enough to warrant accomodating it. Putting It All Together

The result of all these applied principles is a fully responsive web page. Media Queries dictate the main device experinces by applying different sections of the stylesheet, while flexible grids and media optimize the page within those frameworks for any screen size. Replacing pixel values with rems further optimizies the viewing experince for various devices, while other tools simply give you greater options and flexibility in designing the various device experinces according to your designated breakpoints.
